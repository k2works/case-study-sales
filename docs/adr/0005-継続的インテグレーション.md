## 決定

継続的インテグレーションを導入する。理由は以下の通り。

- プロジェクトの品質を保つことができる
- プロジェクトのリスクを軽減できる
- プロジェクトの信頼性を向上できる

## 影響

ポジティブ:
- プロジェクトの品質を保つことができる

ネガティブ:
- 構築の時間がかかる

## コプライアンス

GitHub Actionsを使用する。
